Basics

Notes taken from Jon Duckett’s Book “HTML & CSS: Design and Build Websites”

Intro

HTML describes the structure of web pages and it stands for HyperText Markup Language. The HyperText part refers to the fact that HTML allows you to create links from one page to another easily.
A markup language allows you to annotate text, and these annotations provide additional meaning to the contents of a document

tag, element

The HTML code that are written within tags, which are the components that contain opening and closing angle brackets (< , >), are called HTML elements. Elements are usually made up of these two tags (opening and closing), but may also just contain one.

The terms “tag” and “element” are often used interchangeably. Strictly speaking, however, an element comprises the opening tag and the closing tag and any content that may lie between them.

For example:

This is a paragraph. 

<p> This is a paragraph. </p>

This whole thing can be considered an element. while <p> and </p> are considered to be tags.

———

attribute

Attributes tell us more about elements. They provide additional information about the contents of an element. They are made up of two parts: A name and a value.

For example:

This paragraph is in English 

<p lang="en-us"> This paragraph is in English </p>

The attribute name indicates what kind of extra information you are supplying about the element’s content. It is written in lowercase. In the example above, lang is the name of the attribute. The value is the information or setting for the attribute. It should be placed in double quotes. In this case “en-us” is the value indicating that the language of the text is in US-English. Remember that there is an (=) sign between the name and value, and that it is written inside the opening tag of the element.

Most attribute values are either pre-defined or follow a stipulated format. Check here for a list of all attribute names, along with their corresponding values and formats.

<head>, <body>, <title>

<head> : The <head> element contains information about the page.
<body> : Everything in the <body> element is shown inside the main browser window.
<title> : The content inside the <title> is displayed on the top of the browser, usually on the title bar or tab of the webpage. The title of this page is “Basics”.

Semantic and Structural Markup

Structural Markup: the elements that you can use to describe both headings and paragraphs.
Semantic Markup: provides extra information; such as where emphasis is placed in a sentence. The reason to use these elements is not to affect the appearance, but because there are programs such as screen readers or search engines that use this extra information. A screen reader may add emphasis to a word when reading it out loud to a user.

<b>

<i>

By enclosing words in the tags <i> and </i> we can make characters appear italic. The <i> element also represents a section of text that would be said in a different way from surrounding content - such as technical terms, names of ships, foreign words, thoughts, or other terms that would usually be italicized.

For example:

This is italicized text 

<i> This is italicized text </i>

<sup>, <sub>

The <sup> element is used to contain characters that should be superscript such as suffixes of dates or mathematical concepts like raising a number to a power such as 22.

For example:

On the 4th of September you will learn about E=MC2. 

On the 4<sup>th</sup> of September you will learn about E=MC<sup>2</sup>.

The <sup> element is used to contain characters that should be subscript. It is commonly used with foot notes or chemical formulas such as H2O.

For example:

The amount of CO2 in the atmosphere grew by 2ppm in 20091. 

The amount of CO<sub>2</sub> in the atmosphere grew by 2ppm in 2009<sub>1</sub>.

white space collapsing:

When the browser comes across two or more spaces next to each other, it only displays one space. Similarly if it comes across a line break, it treats that as a single space too. This is known as white space collapsing. Some web page authors take advantage of white space collapsing to indent their code in order to make it easier to follow.

For example:

The moon is drifting away from Earth.

The moon is drifting away from Earth.

<strong>, <em>

The use of the <strong> element indicates that its content has strong importance. For example, the words contained in this element might be said with strong emphasis.

By default, browsers will show the contents of a <strong> element in bold.

For example:

Beware: Pickpockets operate in this area.

This toy has many small pieces and is not suitable for children under five years old. 

<p><strong>Beware:</strong> Pickpockets operate in this area.</p>
<p>This toy has many small pieces and is <strong>not suitable for children under five years old.</strong></p>

The <em> element indicates emphasis that subtly changes the meaning of a sentence.

By default, browsers will show the contents of an <em> element in italic.

For example:

I think Ivy was the first.

I think Ivy was the first.

I think Ivy was the first. 

<p>I <em>think</em> Ivy was the first.</p>
<p>I think <em>Ivy</em> was the first.</p>
<p>I think Ivy was the <em>first</em>.</p>

<blockquote>, <q>

The <blockquote> element is used for longer quotes that take up an entire paragraph. Note how the <p> element is still used inside the <blockquote> element.

Browsers tend to indent the contents of the <blockquote> element, however you should not use this element just to indent a piece of text — rather you should achieve this effect using CSS.

The <q> element is used for shorter quotes that sit within a paragraph.

Both elements may use the cite attribute to indicate where the quote is from. It’s value should be a URL that will have more information about the source of the quotation.

For example:

Did you ever stop to think, and forget to start again?

As A.A. Milne said. Some people talk to animals. Not many listen though. That’s the problem 

<blockquote cite="http://en.wikipedia.org/wiki/Winnie-the-Pooh">
<p>Did you ever stop to think, and forget to start again?</p>
</blockquote>

<abbr>

If you use an abbreviation or an acronym, then the <abbr> element can be used. A title attribute on the opening tag is used to specify the full term.

In HTML4 there was a separate <acronym> element for acronyms. To spell out the full form of the acronym, it also used the title attribute. In HTML5, we use the <abbr> element for both abbreviations and acronyms.

For example:

Prof Stephen Hawking is a theoretical physicist and cosmologist. 

<p><abbr title="Professor">Prof</abbr> Stephen Hawking is a theoretical physicist and cosmologist.</p>

<cite>

When you are referencing a piece of work such as a book, film or research paper, the <cite> element can be used to indicate where the citation is from.
Browsers will render the content of a <cite> element in italics.

For example:

A Brief History of Time by Stephen Hawking has sold over ten million copies worldwide. 

<p><cite>A Brief History of Time</cite> by Stephen Hawking has sold over ten million copies worldwide.</p>

<dfn>

The first time you explain some new terminology (perhaps an academic concept or some jargon) in a document, it is known as the defining instance of it.

The <dfn> element is used to indicate the defining instance of a new term. Some browsers show the <dfn> element in italics.

For example:

A black hole is a region of space from which nothing, not even light, can escape. 

<p>A <dfn>black hole</dfn> is a region of space from which nothing, not even light, can escape.</p>

<address>

The <address> element’s use is to contain contact details for the author of the page.
It can contain a physical address, but it does not have to. For example, it may also contain a phone number or email address.

Browsers often display the content of the <address> element in italics.

For example:

742 Evergreen Terrace, Springfield.

<ins>, <del>, <s>

The <ins> element can be used to show that has been inserted into a document, while the <del> element can show text that has been deleted from it.
The content of an <ins> element is usually underlined, while the content of a <del> element usually has a line through it.

For example:

It was the worst best idea she had ever had. 

<p>It was the <del>worst</del> <ins>best</ins> idea she had ever had.</p>

The <s> element indicates something that is no longer accurate or relevant (but that should not be deleted)

Visually the content of an <s> element will usually be displayed with a line through the center.

For example:

Links

There are several types of links:

  • Links from one website to another
  • Links from one page to another on the same website
  • Links from one part of the web page to another part of the same page
  • Links that open in a new browser window
  • Links that start up your email program and address a new email to someone

<a>

Links are created using the <a> element with the href attribute. When you link to a different website, you use the absolute URL. URL stands for Uniform Resource Locator.

For example:

Metacritic 

<a href="http://www.rottentomatoes.com">Metacritic</a>
linking to other pages on the same site:

When linking to another page on the same site, you do not need to specify the domain name in the URL. You can use a shorthand known as a relative URL. If all the pages of the site are in the same folder, then the value of the href attribute is just the name of the file.

For example:

  • Home
  • About
  • Work
  • Contact 
<ul>
  <li><a href="index.html">Home</a></li>
  <li><a href="about.html">About</a></li>
  <li><a href="work.html">Work</a></li>
  <li><a href="contact.html">Contact</a></li>
</ul>
email links:

To create an email link you start similarly with the <a> element and the href attribute but instead of an absolute or relative URL you enter mailto: followed by an email.

For example:
Email Juan

<a href="mailto:juaneduardoflores@gmail.com">Email Juan</a>
opening links in a new window:

If you want a link to open in a new window, you can use the target attribute on the opening <a> tag. The value of this attribute should be _blank.

For example:

Internet Movie Database 

<a href="http://www.imbd.com" target="_blank">Internet Movie Database</a>
linking to a specific part of the same page:

This page has these links on the top left side that take you to specific parts of this page. The way this is achieved is by using the id attribute. You can use this on any HTML element. In this page’s case, there is an id for every 2nd, 3rd, or 4th level header.

The value of the id attribute should start with a letter or an underscore (not a number or any other character) and, on a single page, no two id attributes should have the same value.

To link to an element that uses the id attribute you use the <a> element again, but the value of the href attribute starts with the # symbol, followed by the value of the id attribute of the element you want to link to.

For example:

basics 

<!-- takes you to the top header -->
<a href="#basics">basics</a>

Images

<img>

To add an image into the page you need to use the <img> element. This is an empty element (which means there is no closing tag). It must carry two attributes: src and alt.

The src attribute tells the browser where it can find the image file. This will usually be a relative URL pointing to an image on your own site. For example, it could be located in a folder called “images”.

The alt attribute provides a text description of the image which describes the image if you cannot see it.

There is also the optional title attribute to provide additional information about the image. Most browsers will display the content of this attribute in a tooltip when the users hovers over the image.

height & width of images:

You can specify the size of the image by providing the dimensions using the width and height attributes.

For example:

<p style="text-align:center;"><img src="images/woof.bmp" alt="A pic of a dog" title="woof!" width="200" height="400"/></p>

Forms

Forms are used to collect information from visitors to your site.

<form>

Form controls live inside a <form> element. The element should always carry the action attribute and will usually have a method and id attribute too.

action

The value for this attribute is the URL for the page on the server that will receive the information in the form when it is submitted.

method

Forms can be sent using one of the two methods: get or post.

With the get method, the values from the form are added to the end of the URL specified in the action attribute. The get method is ideal for:

  • short forms (such as search boxes)
  • when you are just retrieving data from the web server (not sending information that should be added to or deleted from a database)

With the post method the values are sent in what are know as HTTP headers. As a rule of thumb you should use the post method if your form:

  • allows users to upload a file
  • is very long
  • contains sensitive data (e.g. passwords)
  • adds information to, or deletes information from, a database

If the method attribute is not used, the form data will be sent using the getmethod.

For example:

This is where the form controls will apear 

<form action="http://www.example.com/subscribe.php method="get>
  <p>This is where the form controls will apear</p>
</form>

<input>

The <input> element is used to create several different form controls. The value of the type attribute determines what kind of input they will be creating.

type=“text”

When the type attribute is equal to “text”, it creates a singe-line text input.

name

The name attribute is required and is used to let the server know which form control each piece of data was entered into. The value of this attribute identifies the form control and is sent along with the information they enter to the server.

maxlength

The maxlength attribute is optional and is used to limit the number of characters a user may enter into the text field.

For example:

Username: 

<form action="http://www.example.com/login.php">
  <p>Username:
    <input type="text" name="username" size="15" maxlength="30" /> 
  </p>
</form>
password input:

If you want a password input, you can give the type attribute the value of “password”. The text box acts like a single-line text input, except the characters are blocked out.

<textarea>

The <textarea> element is used to create a multi-line text input. Unlike other input elements this is not an empty element. It should therefore have an opening and a closing tag.

For example:

What did you think of this gig? 

<form action="http://www.example.com/comments.php">
  <p>What did you think of this gig?</p>
  <textarea name="comments">Enter your comments...</textarea>
</form>
radio button:

To create a radio button, add the “radio” value to the type attribute.

The values of the name and value attributes are sent to the server when the user selects and submits the form.

the checked attribute can be used to indicate which value (if any) should be selected when the page loads. Only one radio button in a group should use this attribute.

For example:

Please select your favorite genre:


Rock Pop Jazz 
<form action="http://www.example.com/profile.php">
  <p>Please select your favorite genre:</p>
  <br />
  <input type="radio" name="genre" value="rock" checked="checked">Rock</input>
  <input type="radio" name="genre" value="pop">Pop</input>
  <input type="radio" name="genre" value="jazz">Jazz</input>
</form>
checkbox:

To use a checkbox, enter the “checkbox” value to the type attribute.

The name, value, and checked attributes behave similarly to the radio button.

For example:

Please select your favorite music service(s):
iTunes Last.fm Spotify 

<form action="http://www.example.com/profile.php">
  <p>Please select your favorite music service(s):
    <br />
    <input type="checkbox" name="service" value="itunes" checked="checked"> iTunes</input>
    <input type="checkbox" name="service" value="lastfm"> Last.fm</input>
    <input type="checkbox" name="service" value="spotify"> Spotify</input>
  </p>
</form>

<select>

A drop down list box (also known as a select box) allows users to select one option from a drop down list. A <select> element is used to create it. It contains two or more <option> elements.

<option>

The <option> element is used to specify the options that the user can select from.

selected

The selected attribute can be used to indicate the option that should be selected when the page loads. If not used, the first option will be shows when the page loads.

For example:

What device do you listen to music on? 

<form action="http://www.example.com/profile.php">
  <p> What device do you listen to music on? </p>
    <select name="devices">
      <option value="ipod">iPod</option> 
      <option value="radio">Radio</option> 
      <option value="Computer">Computer</option> 
    </select>
</form>
multiple select box:

You can turn a drop down select box into a box that shows more than one option by adding the size attribute. Its value should be the number of options you want to show at once.

You can also allow users to select multiple options from this list by adding the multiple attribute with a value of “multiple”. It is a good idea to tell users if they can select more than one option at a time, and that to select multiple options you use the control key on PC and the command key on Mac.

For example:

Do you play any of the following instruments? (You can select more than one option) 

<form action="http://www.example.com/profile.php">
  <p>Do you play any of the following instruments? (You can select more than one option)</p>
  <select name="instruments" size="3" multiple="multiple">
    <option value="guitar" selected="selected">Guitar</option>
    <option value="drums">Drums</option>
    <option value="keyboard">Keyboard</option>
    <option value="bass">Bass</option>
  </select>
</form>
file input box:

If you want to allow users to upload a file you will need a file input box. To use it, use the “file” value for the type attribute. When you are allowing users to upload files, the method attribute must have a value of “post”.

For example:

Upload your song in MP3 format: 


<form action="http://www.example.com/upload.php" method="post">
  <p>Upload your song in MP3 format:</p>
  <input type="file" name="user-song" /><br />
  <input type="submit" value="Upload" />
</form>
submit button:

To use a submit button, enter the “submit” value to the type attribute. The submit button is used to send a form to the server. It can use a name attribute but it does need to have one. The value attribute is used to control the text htat appears on a button.

For example:

Subscribe to our email list: 

<form action="http://www.example.com/subscribe.php">
  <p>Subscribe to our email list:</p>
  <input type="text" name="email" />
  <input type="submit" name="subscribe" value="subscribe" />
</form>

If you want to use an image for the submit button you can give the type attribute the value of “image”. The src, width, height, and alt attributes work the same way as they do with the <img> element.

For example:

Subscribe to our email list: 

<form action="http://www.example.org/subscribe.php">
  <p>Subscribe to our email list:</p>
  <input type="text" name="email" />
  <input type="image" src="images/subscribe.png" />
</form>

<button>

The <button> element was introduced to allow users moer control over how their buttons appear, and to allow other elements to appear inside the button. This means you can combine text and images.

If you put the “hidden” value to the type attribute, then it will hide the element from the webpage. In the following example the input element is hidden.

For example:

<form action="http://www.example.com/add.php">
  <button> <img src="images/subscribe.png" alt="add" /> Add</button>
  <input type="hidden" name="bookmark" value="lyrics" />
</form>

<label>

Each form control should have its own <label> element as this makes the form accessible to vision-impaired users. There are two ways the label attribute can be used as demonstrated in the following example.

The for attribute states which form control the label belongs to. The value of the id attribute is used to uniquely identify an element from all other elements on a page.

For example:


Gender: 

<label>Age: <input type="text" name="age"/></label>
<br />
Gender:
<input id="female" type="radio" name="gender" value="f">
<label for="female">Female</label>
<input id="male" type="radio" name="gender" value="m">
<label for="male">male</label>
grouping form elements:

<fieldset>, <legend>

You can group related form controls together inside a <fieldset> element. This is useful for longer forms. The <legend> element can come directly after the opening <fieldset> tag and contains a caption which helps identify the purpose of that group of form controls.

For example:

Contact details 


<fieldset>
  <legend>Contact details</legend>
  <label>Email:<br />
  <input type="text" name="email" /></label><br />
  <label>Mobile:<br />
  <input type="text" name="mobile" /></label><br />
  <label>Telephone:<br />
  <input type="text" name="telephone" /></label><br />
</fieldset>
form validation:

Form validation is when a message appears from a form control saying that it has not been filled in correctly. Traditionally, form validation has been performed using JavaScript but HTML5 introduced validation and leaving the work to the browser.
Validation helps ensure the user enter information in a form that the server will be able to understand when the form is submitted.

The required attribute is used with a value of “required” to make a form control required.

For example:



<form action="http://www.example.com/login/" method="post">
  <label for="username">Username:</label>
  <input type="text" name="username" required="required" /><br />
  <label for="password">Password:</label>
  <input type="password" name="password" required="required" /><br />
  <input type="submit" value="Submit" />
</form >
