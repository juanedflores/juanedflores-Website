Basics

Notes taken from Jon Duckett’s Book “HTML & CSS: Design and Build Websites”

Intro

HTML describes the structure of web pages and it stands for HyperText Markup Language. The HyperText part refers to the fact that HTML allows you to create links from one page to another easily.
A markup language allows you to annotate text, and these annotations provide additional meaning to the contents of a document

tag, element

The HTML code that are written within tags, which are the components that contain opening and closing angle brackets (< , >), are called HTML elements. Elements are usually made up of these two tags (opening and closing), but may also just contain one.

The terms “tag” and “element” are often used interchangeably. Strictly speaking, however, an element comprises the opening tag and the closing tag and any content that may lie between them.

For example:

This is a paragraph. 

<p> This is a paragraph. </p>

This whole thing can be considered an element. while <p> and </p> are considered to be tags.

———

attribute

Attributes tell us more about elements. They provide additional information about the contents of an element. They are made up of two parts: A name and a value.

For example:

This paragraph is in English 

<p lang="en-us"> This paragraph is in English </p>

The attribute name indicates what kind of extra information you are supplying about the element’s content. It is written in lowercase. In the example above, lang is the name of the attribute. The value is the information or setting for the attribute. It should be placed in double quotes. In this case “en-us” is the value indicating that the language of the text is in US-English. Remember that there is an (=) sign between the name and value, and that it is written inside the opening tag of the element.

Most attribute values are either pre-defined or follow a stipulated format. Check here for a list of all attribute names, along with their corresponding values and formats.

<head>, <body>, <title>

<head> : The <head> element contains information about the page.
<body> : Everything in the <body> element is shown inside the main browser window.
<title> : The content inside the <title> is displayed on the top of the browser, usually on the title bar or tab of the webpage. The title of this page is “Basics”.

Semantic and Structural Markup

Structural Markup: the elements that you can use to describe both headings and paragraphs.
Semantic Markup: provides extra information; such as where emphasis is placed in a sentence. The reason to use these elements is not to affect the appearance, but because there are programs such as screen readers or search engines that use this extra information. A screen reader may add emphasis to a word when reading it out loud to a user.

<h1>, <h2>, <h3>, <h4>, <h5>, <h6>

HTML has six “levels” of headings. <h1> is used for main headings. <h2> is used for subheadings. As you go on, the heading font size gets smaller.

For example:

This is a subheading of level 2 

<h2> This is a subheading of level 2 </h2>

This is a subheading of level 3

This is a subheading of level 4 

<h4> This is a subheading of level 4 </h4>
This is a subheading of level 5 
<h5> This is a subheading of level 5 </h5>
This is a subheading of level 6 
<h6> This is a subheading of level 6 </h6>

<b>

By enclosing words in the tags <b> and </b> we can make characters appear bold. For example:

This is bold text 

<b> This is bold text </b>

———

<i>

By enclosing words in the tags <i> and </i> we can make characters appear italic. The <i> element also represents a section of text that would be said in a different way from surrounding content - such as technical terms, names of ships, foreign words, thoughts, or other terms that would usually be italicized.

For example:

This is italicized text 

<i> This is italicized text </i>

———

<sup>, <sub>

The <sup> element is used to contain characters that should be superscript such as suffixes of dates or mathematical concepts like raising a number to a power such as 22.

For example:

On the 4th of September you will learn about E=MC2. 

On the 4<sup>th</sup> of September you will learn about E=MC<sup>2</sup>.

The <sup> element is used to contain characters that should be subscript. It is commonly used with foot notes or chemical formulas such as H2O.

For example:

The amount of CO2 in the atmosphere grew by 2ppm in 20091. 

The amount of CO<sub>2</sub> in the atmosphere grew by 2ppm in 2009<sub>1</sub>.

white space collapsing:

When the browser comes across two or more spaces next to each other, it only displays one space. Similarly if it comes across a line break, it treats that as a single space too. This is known as white space collapsing. Some web page authors take advantage of white space collapsing to indent their code in order to make it easier to follow.

For example:

The moon is drifting away from Earth.

The moon is drifting away from Earth.

The moon is drifting away from Earth 

<p> The moon is drifting away from Earth. </p>
<p> The moon            is drifting away from Earth.</p>
<p> The moon is drifting away from

          Earth</p>

<br />

If you wanted to add a line break inside the middle of a paragraph you can use the line break tag <br />.

For example:

The Earth
gets one hundred tons heavier every day
due to falling space dust. 

<p>The Earth<br /> gets one hundred tons heavier every day <br /> due to falling space dust.</p>

———

<hr />

To create a break between themes — such as a change of topic in a book or a new scene in a play — you can add a horizontal rule between sections using the <hr />. tag.

For example:

Venus is the only planet that rotates clockwise.

Jupiter is bigger than all the other planets combined. 

<p>Venus is the only planet that rotates clockwise.</p>
<hr />
<p>Jupiter is bigger than all the other planets combined.</p>

<strong>, <em>

The use of the <strong> element indicates that its content has strong importance. For example, the words contained in this element might be said with strong emphasis.

By default, browsers will show the contents of a <strong> element in bold.

For example:

Beware: Pickpockets operate in this area.

This toy has many small pieces and is not suitable for children under five years old. 

<p><strong>Beware:</strong> Pickpockets operate in this area.</p>
<p>This toy has many small pieces and is <strong>not suitable for children under five years old.</strong></p>

The <em> element indicates emphasis that subtly changes the meaning of a sentence.

By default, browsers will show the contents of an <em> element in italic.

For example:

I think Ivy was the first.

I think Ivy was the first.

I think Ivy was the first. 

<p>I <em>think</em> Ivy was the first.</p>
<p>I think <em>Ivy</em> was the first.</p>
<p>I think Ivy was the <em>first</em>.</p>

———

<blockquote>, <q>

The <blockquote> element is used for longer quotes that take up an entire paragraph. Note how the <p> element is still used inside the <blockquote> element.

Browsers tend to indent the contents of the <blockquote> element, however you should not use this element just to indent a piece of text — rather you should achieve this effect using CSS.

The <q> element is used for shorter quotes that sit within a paragraph.

Both elements may use the cite attribute to indicate where the quote is from. It’s value should be a URL that will have more information about the source of the quotation.

For example:

Did you ever stop to think, and forget to start again?

As A.A. Milne said. Some people talk to animals. Not many listen though. That’s the problem 

<blockquote cite="http://en.wikipedia.org/wiki/Winnie-the-Pooh">
<p>Did you ever stop to think, and forget to start again?</p>
</blockquote> 
<p>As A.A. Milne said. <q>Some people talk to animals. Not many listen though. That's the problem</q></p>

———

<abbr>

If you use an abbreviation or an acronym, then the <abbr> element can be used. A title attribute on the opening tag is used to specify the full term.

In HTML4 there was a separate <acronym> element for acronyms. To spell out the full form of the acronym, it also used the title attribute. In HTML5, we use the <abbr> element for both abbreviations and acronyms.

For example:

Prof Stephen Hawking is a theoretical physicist and cosmologist. 

<p><abbr title="Professor">Prof</abbr> Stephen Hawking is a theoretical physicist and cosmologist.</p>

———

<cite>

When you are referencing a piece of work such as a book, film or research paper, the <cite> element can be used to indicate where the citation is from.
Browsers will render the content of a <cite> element in italics.

For example:

A Brief History of Time by Stephen Hawking has sold over ten million copies worldwide. 

<p><cite>A Brief History of Time</cite> by Stephen Hawking has sold over ten million copies worldwide.</p>

———

<dfn>

The first time you explain some new terminology (perhaps an academic concept or some jargon) in a document, it is known as the defining instance of it.

The <dfn> element is used to indicate the defining instance of a new term. Some browsers show the <dfn> element in italics.

For example:

A black hole is a region of space from which nothing, not even light, can escape. 

<p>A <dfn>black hole</dfn> is a region of space from which nothing, not even light, can escape.</p>

———

<address>

The <address> element’s use is to contain contact details for the author of the page.
It can contain a physical address, but it does not have to. For example, it may also contain a phone number or email address.

Browsers often display the content of the <address> element in italics.

For example:

homer@example.org

742 Evergreen Terrace, Springfield. 

<address><p><a ahref="mailto:homer@example.org">homer@example.org</a></p>
<p>742 Evergreen Terrace, Springfield.</p>

———

<ins>, <del>, <s>

The <ins> element can be used to show that has been inserted into a document, while the <del> element can show text that has been deleted from it.
The content of an <ins> element is usually underlined, while the content of a <del> element usually has a line through it.

For example:

It was the worst best idea she had ever had. 

<p>It was the <del>worst</del> <ins>best</ins> idea she had ever had.</p>

The <s> element indicates something that is no longer accurate or relevant (but that should not be deleted)

Visually the content of an <s> element will usually be displayed with a line through the center.

For example:

Laptop computer:

Was $995

Now only $375 

<p>Laptop computer:</p>
<p><s>Was $995</s></p>
<p>Now only $375</p>

Lists

<li>

Each item in a list is placed between the opening <li> tag and its closing </li> tag. The li stands for “list item”.

Browsers indent lists by default.

<ol>

An ordered list is created with the <ol> element.

For example:
  1. Chop potatoes into quarters
  2. Simmer in salted water for 15-20 minutes until tender
  3. Heat milk, butter and nutmeg
  4. Drain potatoes and mash
  5. Mix in the milk mixture 
<ol>
  <li>Chop potatoes into quarters</li>
  <li>Simmer in salted water for 15-20 minutes until tender</li>
  <li>Heat milk, butter and nutmeg</li>
  <li>Drain potatoes and mash</li>
  <li>Mix in the milk mixture</li>
</ol>

<ul>

An unordered list is created with the <ul> element.

For example:
  • 1kg King Edward potatoes
  • 100ml milk
  • 50g salted butter
  • Freshly grated nutmeg
  • Salt and pepper to taste 
<ul>
  <li>1kg King Edward potatoes</li>
  <li>100ml milk</li>
  <li>50g salted butter</li>
  <li>Freshly grated nutmeg</li>
  <li>Salt and pepper to taste</li>
</ul>

The <dt> element is used to contain the term being defined, while <dd> is used to contain the definition.

Links

There are several types of links:

  • Links from one website to another
  • Links from one page to another on the same website
  • Links from one part of the web page to another part of the same page
  • Links that open in a new browser window
  • Links that start up your email program and address a new email to someone

<a>

Links are created using the <a> element with the href attribute. When you link to a different website, you use the absolute URL. URL stands for Uniform Resource Locator.

For example:

Metacritic 

<a href="http://www.rottentomatoes.com">Metacritic</a>
linking to other pages on the same site:

When linking to another page on the same site, you do not need to specify the domain name in the URL. You can use a shorthand known as a relative URL. If all the pages of the site are in the same folder, then the value of the href attribute is just the name of the file.

For example:

  • Home
  • About
  • Work
  • Contact 

<p>
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="work.html">Work</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</p>
email links:

To create an email link you start similarly with the <a> element and the href attribute but instead of an absolute or relative URL you enter mailto: followed by an email.

For example:
Email Juan

<a href="mailto:juaneduardoflores@gmail.com">Email Juan</a>
opening links in a new window:

If you want a link to open in a new window, you can use the target attribute on the opening <a> tag. The value of this attribute should be _blank.

For example:
Internet Movie Database

<a href="http://www.imbd.com" target="_blank">Internet Movie Database</a>
linking to a specific part of the same page:

This page has these links on the top left side that take you to specific parts of this page. The way this is achieved is by using the id attribute. You can use this on any HTML element. In this page’s case, there is an id for every 2nd, 3rd, or 4th level header.

The value of the id attribute should start with a letter or an underscore (not a number or any other character) and, on a single page, no two id attributes should have the same value.

To link to an element that uses the id attribute you use the <a> element again, but the value of the href attribute starts with the # symbol, followed by the value of the id attribute of the element you want to link to.

For example:
basics - takes you to the top header with the id “basics”.

<a href="#basics">basics</a>
